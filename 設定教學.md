# FinLedger 私人存取設定教學
## Cloudflare Zero Trust Access + iPhone PWA + 資料安全

---

## 目標

完成後你的 FinLedger 將會：
- 有一個 `https://finledger-xxxx.vercel.app` 的專屬網址
- 只有你能登入（用你的 Email 驗證）
- 在 iPhone 上像 App 一樣使用（加入主畫面）
- 資料安全備份在 Google Sheet

全部免費，不需要寫任何程式碼。

---

## 第一部分：部署到 Vercel（如果還沒做）

### 步驟 1：上傳到 GitHub

1. 開啟 https://github.com/new
2. Repository name 填 `finledger`，選 **Public**，點 **Create repository**
3. 點「uploading an existing file」
4. 把 **index.html** 和 **manifest.json** 兩個檔案都拖進去
5. 點 **Commit changes**

### 步驟 2：部署到 Vercel

1. 開啟 https://vercel.com，用 GitHub 帳號登入
2. 點 **Add New → Project**
3. 找到 `finledger` repo，點 **Import**
4. 所有設定保持預設，點 **Deploy**
5. 約 30 秒後取得你的網址：`https://finledger-xxxx.vercel.app` ✅

---

## 第二部分：Cloudflare Access（只有你能登入）

### 步驟 1：建立 Cloudflare 帳號

1. 開啟 https://cloudflare.com
2. 點右上角 **Sign Up**，用你的 Email 免費註冊
3. 登入後，點左側選單的 **Zero Trust**
   - 如果找不到，搜尋「Zero Trust」

### 步驟 2：設定 Zero Trust

1. 進入 Zero Trust 後，系統可能要你選方案 → 選 **Free**（免費，每月最多 50 位使用者，你只需要 1 位）
2. 輸入一個 Team Name（例如你的名字縮寫），點 **Next**

### 步驟 3：建立 Application

1. 左側選單點 **Access → Applications**
2. 點右上角 **Add an application**
3. 選 **Self-hosted**（自托管）
4. 填入以下資料：
   - **Application name**：`FinLedger`
   - **Session Duration**：`1 month`（一個月免重新登入）
   - **Application domain**：填你的 Vercel 網址，例如 `finledger-xxxx.vercel.app`
     - Subdomain 填：`finledger-xxxx`
     - Domain 填：`vercel.app`
5. 點 **Next**

### 步驟 4：設定存取政策（只允許你的 Email）

1. Policy Name 填：`Only Me`
2. Action 選：**Allow**
3. Configure rules 區塊：
   - Include → **Emails** → 填入你自己的 Email 地址
4. 點 **Next** → **Add application**

### 步驟 5：測試登入

1. 打開你的 Vercel 網址
2. 應該會看到 Cloudflare 的登入畫面
3. 輸入你的 Email → 收到驗證碼 → 輸入驗證碼
4. 成功進入 FinLedger！

> **注意**：每次登入有效期 1 個月，到期後需要重新驗證 Email

## 第三部分：iPhone 加入主畫面（PWA App）

### 步驟 1：用 Safari 打開

> ⚠️ 必須用 **Safari** 瀏覽器，Chrome 不支援 iOS PWA 安裝

1. 打開 iPhone 上的 Safari
2. 輸入你的 FinLedger 網址
3. 完成 Cloudflare Email 驗證後進入 FinLedger

### 步驟 2：加入主畫面

1. 點 Safari 下方的 **分享按鈕**（方框加箭頭的圖示 ⬆）
2. 向下滑，找到 **「加入主畫面」**
3. 名稱改為 `FinLedger`（或保持不變）
4. 點右上角 **「新增」**

### 步驟 3：從主畫面打開

1. 回到 iPhone 主畫面
2. 找到 FinLedger 的圖示（深藍底綠色折線圖）
3. 點開後是全螢幕模式，沒有網址列，就像真正的 App！

> **離線支援**：即使沒有網路，也可以查看已載入的資料（無法更新股價）

## 第四部分：資料安全保護

### 資料儲存位置

| 位置 | 說明 | 風險 |
|------|------|------|
| 瀏覽器 localStorage | 本機即時儲存 | 清除瀏覽器資料會消失 |
| Google Sheet | 雲端備份 | 需要有網路才能同步 |
| JSON 匯出檔 | 本地備份 | 需要手動匯出 |

### 建議的資料保護習慣

**每天**：FinLedger 會自動在背景同步到 Google Sheet（若已連結）

**每週**：FinLedger 會在超過 7 天未備份時顯示橘色提醒橫幅，點「立即同步到 Sheet」即可

**換裝置時**：
1. 在新裝置開啟 FinLedger
2. 去「設定」頁面，填入相同的 Apps Script URL
3. 點「從 Sheet 匯入資料」
4. 所有資料恢復完畢 ✅

### Google Sheet 的資料加密

你的 Google Sheet 受 Google 帳號保護：
- 使用 Google 的 AES-256 加密儲存
- 只有你的 Google 帳號可以存取
- 建議開啟 Google 帳號的兩步驟驗證

### 如何開啟 Google 兩步驟驗證

1. 開啟 https://myaccount.google.com/security
2. 找到「兩步驟驗證」，點「開始使用」
3. 選擇驗證方式（推薦：Google 驗證應用程式）

## 常見問題

**Q：Cloudflare 免費方案夠用嗎？**
A：完全夠用。免費方案支援最多 50 位使用者/月，你只需要 1 位。

**Q：每次都要輸入 Email 驗證碼嗎？**
A：不用。設定為 1 個月有效期，每個月只需要驗證一次。

**Q：Vercel 網址太長了，可以改短嗎？**
A：可以。在 Vercel 專案設定 → Domains → 新增自訂網域，但需要你有自己的網域名稱（如 finance.yourdomain.com）。

**Q：Google Sheet 同步失敗怎麼辦？**
A：先用「匯出 JSON 備份」保存資料，再重新部署 Apps Script（參考 FinLedger 設定頁的安裝步驟）。

**Q：如果我不想用 Cloudflare，有其他方式嗎？**
A：有，但都需要寫程式碼（如 Vercel Edge Middleware 加 Basic Auth）。Cloudflare 是最簡單的免費方案。
